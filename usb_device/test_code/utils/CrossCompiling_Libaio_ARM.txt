Steps to get usb usermode driver running with libaio:

1. Get the libaio source tarball

Version	: 0.3.106	
Download location: http://ftp.de.debian.org/debian/pool/main/liba/libaio/libaio_0.3.106.orig.tar.gz
Filename : libaio_0.3.106.orig.tar.gz

2. Get the patch that adds more architectures

Download link from : http://www.hadrons.org/~guillem/patches.html
Download location  : http://www.hadrons.org/~guillem/patches/libaio/libaio_0.3.106_new_arches.patch
Filename:	libaio_0.3.106_new_arches.patch

NOTE: There is an alternative patch 00_arches.patch (provided by Debian I think) which I have not been able to locate.
This could be used as well.

3. Apply the patch and make the following modification

In src/syscall-arm.h, line 100, Replace "type ffname ..." with "type fname ..."

type fname(type1 arg1, type2 arg2, type3 arg3, type4 arg4, type5 arg5) {\
  __SYS_REG(sname)

For more info:
http://www.mail-archive.com/debian-bugs-dist@lists.debian.org/msg250918.html
http://lists.linuxtogo.org/pipermail/openembedded-devel/2006-October/000598.html
  
4. Cross-compile the libaio library

 make CC=arm-none-linux-gnueabi-gcc ARCH=arm prefix=`pwd`/usr install
 
 This will generate the following files:
 ./usr/include/libaio.h
 ./usr/lib/libaio.a
 ./usr/lib/libaio.so
 ./usr/lib/libaio.so.1
 ./usr/lib/libaio.so.1.0.1
 
 The libaio.so.1.0.1 file is the actual file and the other two .so files are symlinks to it.
4b. Check the file type of this file to see if it has actually been compiled for ARM.
 	file libaio.so.1.0.1
 	
5. Copy the ./usr directory to a convenient location. Compile the driver with the following command

arm-none-linux-gnueabi-gnueabi-gcc -Wall -DAIO -g -o usb usb.c usbstring.c -lpthread -laio -I /vobs/wtbu/OMAPSW_L/linux/kernel_org/2.6_kernel/include/ -I ./usr/include/ -L ./usr/lib/ -L /vobs/wtbu/OMAPSW_L/linux/kernel_org/2.6_kernel/lib/

This should generate the 'usb' executable. Check the filetype to ensure it is compiled for ARM

6. Copy the libaio.so.1 file to /lib on your target filesystem. This will ensure that the 'usb' executable can find it.
